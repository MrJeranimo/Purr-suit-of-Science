# Contributing

This file is an overview of what you need to know to properly and respectfully contribute to this project.

## Found an Issue?

If you have found an issue specifically with Purr-suit of Science, which means that you have tested that the issue is not a KSA or StarMap issue, follow the guidelines below for submitting the issue.

1) Go to the `Issues` tab for this repository.
2) Create a new issue.
3) Add a descriptive name for the issue.
4) In the description section, make sure to include what the issue is, how to replicate the issue, and your system specs (operating system, CPU, and GPU). Include any other information you deem relevant as well.
5) Add the `Bug` tag to the issue.
6) Click the `Create Issue` button to publish the issue.

By following these steps you can insure that we are able to better respond and possible fix issues that are found by the community.

## Contributing Code or Files

For those of you who want to be able contribute by codeing or adding part files, this section is for you. All contributions should be done using and respecting the branch rulesets applied to them. Follow the steps below to start.

1) Create your own fork of the repository.
2) Create a branch which should be named `[username]/[feature]` where `[username]` is your Github username and `[feature]` is a short, descriptive name of the feature you wish to add.
3) Make your changes to the branch. Make sure to test your changes to the repository.
4) Add all the required files to your new commit. If you had to change the filepaths in the `Purr-suit of Science.csproj` file, please either revert them before the commit or do not add these changes to the commit.
5) Commit to your branch.
6) Pull/merge the commit to your fork of the repository.
7) Create your pull request to the main Purr-suit of Science repository and include a detailed description of the changes. A maintainer will look at your changes and either give you feedback or approve and merge your changes whenever they can.

By following these steps it should allow for smooth and easy contributions to the project.

## Suggesting a feature

If you wish to suggest a feature for the project, you can do that through the issues tab.

1) Go to the `Issues` tab for this repository.
2) Create a new issue.
3) Add a descriptive name for the new feature.
4) In the description section, make sure to include what the feature is and ideas on how to make it (if known). Include any other information you deem relevant as well.
5) Add the `new-feature` and `suggestion` tags to the issue.
6) Click the `Create Issue` button to publish the issue.

This will allow all contributors to easily see the suggestion in the hopes it will be added.
